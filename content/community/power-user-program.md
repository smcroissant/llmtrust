# ⚡ Power User Program — LLM Trust

## Vision
Identifier, récompenser et fidéliser les membres les plus actifs de la communauté. Les Power Users sont nos meilleurs ambassadeurs, testeurs et sources de feedback.

---

## 🎯 Qui est un Power User ?

### Critères quantitatifs (minimum 3 sur 5)
| Critère | Seuil | Mesure |
|---------|-------|--------|
| **Reviews publiées** | 10+ reviews de qualité | Compteur dans le profil |
| **Modèles ajoutés** | 5+ modèles suggérés/approuvés | Via le formulaire de suggestion |
| **Ancienneté** | 3+ mois actif | Date d'inscription |
| **Contributions communauté** | 20+ messages utiles Discord | Ou 5+ PRs merged sur GitHub |
| **Benchmarks partagés** | 3+ benchmarks vérifiés | Uploads validés |

### Critères qualitatifs
- Les reviews sont **détaillées et constructives** (pas juste "c'est bien")
- Interactions **positives** avec la communauté
- **Régularité** — contribue au moins 1x par semaine
- Aide les **nouveaux membres** naturellement

---

## 🏅 Niveaux de Power User

### ⚡ Power User (Niveau 1)
**Comment l'obtenir:** Remplir les critères ci-dessus + candidature ou nomination

**Avantages:**
- Rôle Discord `⚡ Power User` + couleur or
- Accès au channel privé `#power-user-lounge`
- Badge sur le profil LLM Trust
- Mention dans le README (section Contributors)

### 🏆 Ambassador (Niveau 2)
**Comment l'obtenir:** 3+ mois en tant que Power User + contributions exceptionnelles

**Avantages:**
- Tout ce qui est dans Power User
- Rôle Discord `🏆 Ambassador` + couleur violet
- **Early access** aux nouvelles features (beta privée)
- **Feedback sessions privées** mensuelles avec l'équipe
- **Swag pack** (stickers, t-shirt, mug)
- Invitation aux **décisions produit** (sondages internes)

### 🌟 Legend (Niveau 3)
**Comment l'obtenir:** Contributions transformantes (outils, contenu viral, partenariats)

**Avantages:**
- Tout ce qui est dans Ambassador
- Rôle Discord `🌟 Legend`
- **Co-author** sur les blog posts si souhaité
- **Invitation aux events** de l'équipe (meetups, confs)
- **Consulting calls** trimestriels 1-on-1 avec le CEO

## 🔄 Processus

### Identification
1. **Automatique:** Le système track les métriques (reviews, benchmarks, messages Discord)
2. **Nomination:** Un membre de l'équipe ou de la communauté peut nominer quelqu'un
3. **Auto-candidature:** Un formulaire de candidature dans le profil

### Évaluation (hebdomadaire)
- L'équipe Customer Success (Aura) passe en revue les candidats chaque vendredi
- Décision : ✅ accepté, 🔄 demande de plus d'activité, ❌ pas encore
- Notification par DM Discord

### Maintien du statut
- **Activité minimale:** 1 contribution/mois pour maintenir le statut
- **Révision trimestrielle:** vérifier que les Power Users sont toujours actifs
- **Graduation automatique:** inactif 3 mois → retour au rôle standard (avec possibilité de réactivation)

## 🎁 Système de Récompenses

### Immédiates (automatiques)
| Action | Récompense |
|--------|------------|
| 1ère review publiée | Badge "First Review" + 🎉 réaction Discord |
| 10 reviews | Badge "Reviewer" |
| 1er benchmark partagé | Badge "Benchmark Pioneer" |
| 50 messages utiles Discord | Badge "Community Pillar" |
| 1ère PR merged | Badge "Contributor" + shoutout #announcements |

### Par niveau
| Niveau | Récompenses |
|--------|-------------|
| ⚡ Power User | Rôle Discord + badge profil + accès lounge |
| 🏆 Ambassador | Early access + swag + feedback sessions |
| 🌟 Legend | Co-authorship + events + 1-on-1 calls |

### Reconnaissance publique
- **Hall of Fame** dans le profil (visible par tous)
- **Monthly shoutout** dans #announcements ("Power User du mois")
- **Testimonials** sur la landing page (avec permission)

## 💬 Feedback Sessions (Ambassador+)

### Format
- **Fréquence:** Mensuel, 1er mercredi du mois
- **Durée:** 30 minutes
- **Participants:** 4-6 Ambassadors + 2 membres de l'équipe
- **Format:** Visio (Google Meet) ou voice Discord

### Agenda type
1. **What's new** — l'équipe présente les dernières updates (10 min)
2. **Feedback** — les Ambassadors partagent retours et suggestions (15 min)
3. **Sneak peek** — aperçu des features en développement (5 min)

### Compte-rendu
- Résumé posté dans `#power-user-lounge`
- Actions items trackées dans GitHub Issues
- Les Ambassadors reçoivent un récap par email

## 🚀 Lancement du Programme

### Semaine 1
- [ ] Annoncer le programme dans #announcements
- [ ] Ouvrir les auto-candidatures (formulaire Google Form)
- [ ] Nommer 3-5 Power Users fondateurs (early adopters)

### Semaine 2-4
- [ ] Première revue hebdomadaire des candidatures
- [ ] Créer le channel `#power-user-lounge`
- [ ] Setup les badges sur la plateforme

### Mois 2
- [ ] Première feedback session
- [ ] Évaluer et ajuster les critères
- [ ] Annoncer le premier "Power User du mois"

### Principes
1. **Transparent** — les critères sont publics, pas de favoritisme
2. **Accessible** — n'importe qui peut devenir Power User
3. **Valuable** — les récompenses doivent être réellement utiles
4. **Maintained** — le statut se maintient par l'activité, pas par l'historique
